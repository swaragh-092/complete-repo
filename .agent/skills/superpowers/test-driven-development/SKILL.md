# Test Driven Development (TDD)

## Goal

Ensure tests are written before implementation.

## Instructions

1. Write failing test.
2. Implement code until test passes.
3. Refactor.
4. Reject code written *before* tests.

## Output

- Passing tests and implementation with refactors.
